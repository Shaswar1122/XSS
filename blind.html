Cross site scripting - blind

Blind XSS is a persistent type of cross site scripting, where the attacker will blindly deploy malicious payloads onto a vulnerable web page that does not properly sanitise user input, through a user input field, such as a search box, that will then be stored and saved to a database (a persistent state).

What happens next is the crucial part. At this stage, the attacker has no idea where abouts the payload is, when the payload will be executed, or even if the payload will be executed. So, the waiting game begins. The attacker must wait for a victim to trigger the attack; the victims’ browser will remove the payload out of storage, and render it onto a web page, and this is all done by an unsuspecting user.

Once this attack is in action, it can be used to steal sensitive information from unsuspecting users such as: passwords, login credentials, and credit card details.

The most identifiable difference in blind XSS attacks is that the attacker is essentially ‘blind’ when conducting this attack. They don't know if their attack will be executed in a day’s time or a month’s time, in some cases the payload may not ever be executed. For the attack to succeed, the attacker must ensure they have enough payloads deployed, the attacker must also ensure they implement something for the call backs that they may receive from the payloads they previously deployed.



Examples of Blind XSS:

• Third – party integrations: attackers can inject malicious code integrations such as plugins, which will be executed by the website, when the integration is loaded.

• Malicious file uploads: attackers can upload malicious files to a website, which are then executed by users who download/ access the malicious files.

Solutions:

solution 1

• The first one can be validating and sanitising all user input, this includes data that is entered through user input fields, because as we have learnt previously it is through user input fields that attackers deploy malicious payloads.

solution 2

• One way a user can protect themselves is by being cautious when it comes to which web applications they are visiting and if they are secure or not.

solution 3

• Users should also make a note to always keep their browser software up to date with any of the latest security patches.
