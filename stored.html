Cross site scripting - Stored

Stored cross-site scripting (XSS) is a type of web security attack that can lead to devastating consequences. Stored XSS occurs when an attacker uses malicious scripts which are inserted into a web application or website. This malicious script is stored on the web server and is executed on any user that accesses the affected web page.

Stored XSS can lead to severe consequences for both the affected user and the web application, with data theft being the main consequence of stored XSS. The scripts can be used to steal data such as login credentials, personal information, or financial details. Web applications can also be affected by the malicious scripts which can lead to website defacement. This could be in the form of content being modified or redirecting users to a fake website. Overall, the consequences would lead to the website/applications reputation being damaged as users may lose trust in the platforms security and may face legal consequences such as fines or lawsuits due to data breaches.

Examples of Stored XSS:

• Exploiting user input field: attackers can inject malicious scripts into search boxes and comment fields which the web application then stores on the server once the request is sent.

• Exploiting vulnerabilities in the server side: web apps store user data in a database without a proper level of sanitisation, hackers can inject stored XSS scripts.

Solutions:

solution 1

• sanitize user input by validating and filtering out any unusual characters to remove any malicious code.

solution 2

• Developers can also use encoding techniques such as HTML encoding which is a technique to convert special character into their corresponding HTML entities. This ensures that the user input is currently interpreted by the browser and does not execute any malicious scripts.

solution 3

• Implementing Content Security Policies (CSPs) is another counter measure against stored XSS attacks, this restricts the types of scripts executed by a web page. CSPs allows developers to whitelist trusted scripts used by their own servers but blocks any scripts from untrusted sources
