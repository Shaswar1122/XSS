Cross site scripting - DOM

DOM XSS is a type of security vulnerability that takes place in web applications. The attacker injects malicious code into the web page that is executed by the victim’s browser. This is an XSS type of attack which targets the client-side of an applicant, especially the Document Object Model (DOM) but not the server-side.
DOM is used for the representation of the web page’s structure, and it is influenced by JavaScript to make a dynamic and interactive web experience, but this flexibility is what makes it easy for attackers to manipulate it.


Examples of DOM XSS:

• Changing the content of a webpage: For example, the attacker changes the login button and makes you enter all your login details such as the username and the password to a fake login form. Hacking someone’s email login details could give you access to change all their passwords by forgetting the password and changing it by the link sent to the email.
• Using an event handler: The attacker uses event handler to inject malicious code. The attacker could add a mouseover event which will work when the victim hovers over the link, they would be taken to a different fake login page or steal the cookies of their session.



Different ways to prevent DOM XSS:

.

solution 1

• Sanitise user input
Use a library with a built-in sanitation feature to do the sanitation automatically so you do not need to worry about your code being vulnerable to any attacker.

solution 2

• Use an XSS filter
This mechanism analyses the code and identifies any malicious code. This filter can remove or recode any malice from being executed by the browser. The user can include it as an extension to the web browser or have it as part of the web application.



solution 3

• Limit use of JavaScript
This is the easiest way of preventing XSS attacks but if you do not have JavaScript in your application or web it won’t be very interactive for the user.


Video explanation

To find a DOM based XSS vulnerability within a website we need to first scope out the website. Once this is done, we need to find a user input field such as a comment box, or search bar. (This is because malicious scripts need to be placed into an input box). Now we must place the script into the input field, and all that is left to do is wait for a user to interact with the website.
